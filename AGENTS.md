# AGENTS.md

This repository is a markdown-first Russian translation workspace for *The Book of HPMOR Fanfics*.

## Core Workflow

- Translate from `books/various-muggles/source/` into the matching file under `books/various-muggles/output/`.
- Preserve Markdown structure, links, anchors, separators, asset paths, and file names.
- Do not translate directly in `site/src/content/chapters/`; those files are generated by `bun run sync-content`.
- Run:
  - `python3 scripts/sync_manifest.py --book-id various-muggles`
  - `python3 scripts/validate_run.py --book-id various-muggles`
  - `bun run sync-content` from `site/`

## Canon And Style

- Potter's Army / ROSMEN canon naming wins for Harry Potter terms.
- `hpmor.ru` wins for the Russian HPMOR title and chapter-title precedent.
- Prefer natural Russian over literal calques.
- Use `glossary/glossary.md` and `glossary/style.md`.
- If a new canon term, spell, idiom, or title decision appears, update `glossary/glossary.md`.
- If a story is translated for the first time, ensure `books/various-muggles/config.json` has its Russian title under `translated_story_titles`.

## Automation Rules

- When running in a detached worktree, do not manually choose the next segment from `scripts/pending.py`.
- Use `python3 scripts/automation_claim.py claim --book-id various-muggles --automation-id hourly-segment --worktree-root "$PWD" --json`.
- The claim helper excludes non-reader packaging/frontmatter pages such as `cover.xhtml`, `titlepage.xhtml`, `toc.xhtml`, and `preface.xhtml`.
- Translate only the claimed segment.
- Validate in the worktree, then publish through the canonical checkout with `python3 scripts/publish_from_worktree.py`.
- Do not commit or push directly from the detached worktree if the publish helper is available.

## Done Means

- The translated segment exists in `books/various-muggles/output/`.
- `sync_manifest`, `validate_run`, and `bun run sync-content` pass.
- Generated reader files were refreshed by the pipeline.
- If publishing is part of the task, the result is committed and pushed from the canonical checkout.
